# Автоматизований тест Wikipedia (Playwright)

## Опис проєкту

Цей проєкт містить автоматизований тест, який перевіряє зміну мови інтерфейсу зареєстрованим користувачем через персональні налаштування Wikipedia (опис кейсу орієнтований на українську вікі; URL задається в `BASE_URL`).

Тест реалізований з використанням Playwright + TypeScript.

---

## Автоматизований тест-кейс

**ID:** TC-UI-LANG-01  

**Назва:** Зміна мови інтерфейсу зареєстрованим користувачем через сторінку налаштувань

**Середовище:** Українська Wikipedia — [https://uk.wikipedia.org](https://uk.wikipedia.org) (сценарій перевірено на цьому інстансі). Для автоматизованого прогону базовий URL задається змінною `BASE_URL` у `.env`.

**Передумови:**
- Існує дійсний обліковий запис на цільовій вікі.
- На початку сценарію користувач **не** авторизований (вхід виконується в кроках).
- Доступ до сторінки персональних налаштувань не обмежений для цього облікового запису.

**Тестові дані:**
- Облікові дані: валідний логін і пароль (у проєкті — `WIKI_USERNAME`, `WIKI_PASSWORD` у `.env`, у тексті кейсу пароль не зазначати).
- Нова мова інтерфейсу: будь-яке значення зі списку, **відмінне** від поточного на момент кроку вибору (у проєкті можна зафіксувати код мови через `TARGET_LANGUAGE` у `.env`).

**Кроки:**
1. Відкрити головну сторінку цільової вікі (`https://uk.wikipedia.org` при ручній перевірці за цим кейсом).
2. Авторизуватися обліковим записом з тестових даних (типово посилання на вхід у шапці сторінки).
3. У правому верхньому куті відкрити меню облікового запису (клік по імені користувача або аватарі).
4. У меню обрати перехід на сторінку персональних налаштувань. У україномовному інтерфейсі пункт може відображатися як **«Параметри»** або **«Налаштування»** — залежно від скіна та версії; важливо потрапити на `Special:Preferences`.
5. На сторінці налаштувань знайти розділ **«Інтернаціоналізація»** (вкладка з профілем / загальними параметрами, зазвичай відкривається першою).
6. Зафіксувати поточне значення поля мови інтерфейсу; обрати в списку **іншу** мову, ніж поточна.
7. Прокрутити сторінку донизу й натиснути кнопку **«Зберегти»**.

**Очікуваний результат:**
- Збереження проходить без повідомлення про помилку; за наявності — показується підтвердження успішного збереження налаштувань (текст залежить від мови інтерфейсу).
- У розділі «Інтернаціоналізація» в полі вибору мови інтерфейсу відображається **обрана** мова.
- Після оновлення сторінки або переходу на іншу сторінку вікі підписи елементів інтерфейсу (меню, типові кнопки) відповідають **новій** мові інтерфейсу.

**Післяумови (рекомендовано):** повернути мову інтерфейсу до початкового значення тим самим шляхом, щоб наступні перевірки або інші тести не залежали від залишеної мови.

## Як користуватися проєктом (локально)

### Вимоги

- [Node.js](https://nodejs.org/) (LTS рекомендовано)
- npm

### Підготовка

1. **Встановити залежності npm:**

   ```bash
   npm install
   ```

2. **Встановити браузери для Playwright** (один раз після клонування):

   ```bash
   npx playwright install
   ```

3. **Налаштувати змінні середовища.** Скопіюйте `.env.example` у `.env` і вкажіть облікові дані Wikipedia та (за потреби) URL і цільову мову:

   ```bash
   cp .env.example .env
   ```

   У файлі `.env` заповніть `WIKI_USERNAME`, `WIKI_PASSWORD`; за потреби змініть `BASE_URL` та `TARGET_LANGUAGE`.

### Запуск тестів

| Команда | Опис |
|--------|------|
| `npm test` | Запуск усіх тестів у headless-режимі (за замовчуванням) |
| `npm run test:headed` | Той самий запуск, але з видимим вікном браузера |
| `npm run test:ui` | Інтерактивний режим Playwright UI |

Окремий файл або тег:

```bash
npx playwright test tests/change-language.spec.ts
```

### Звіт про прогін (HTML)

- Після виконання тестів Playwright генерує **HTML-звіт** у каталозі **`playwright-report/`** (у корені проєкту).
- Щоб **відкрити останній звіт у браузері**, виконайте:

  ```bash
  npm run report
  ```

  Це еквівалентно `npx playwright show-report` і відкриє збережений звіт з `playwright-report/`.

- За потреби артефакти невдалих прогонів (скріншоти, відео, trace) з’являються в **`test-results/`** згідно з налаштуваннями в `playwright.config.ts`.

## Запуск у Docker (звіт на машині-хості)

Образ збігається з версією Playwright у проєкті (`Dockerfile`).

1. Збірка:

   ```bash
   docker build -t wikipedia-playwright-aqa .
   ```

2. Прогін з монтуванням каталогів, щоб **HTML-звіт і артефакти** одразу з’явилися у вашій локальній папці проєкту (це відповідає вимозі «отримати звіт на хості» без ручного копіювання):

   ```bash
   docker run --rm \
     --env-file .env \
     -v "$(pwd)/playwright-report:/app/playwright-report" \
     -v "$(pwd)/test-results:/app/test-results" \
     wikipedia-playwright-aqa
   ```

   Після завершення відкрийте звіт: `npm run report` або відкрийте `playwright-report/index.html` у браузері.

3. **Якщо потрібен саме `docker cp`:** спочатку запустіть контейнер з іменем без `--rm`, потім скопіюйте каталог:

   ```bash
   docker run --name wiki-aqa-run --env-file .env wikipedia-playwright-aqa
   docker cp wiki-aqa-run:/app/playwright-report ./playwright-report
   docker rm wiki-aqa-run
   ```
